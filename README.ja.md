# 投資デューデリジェンス司令室

[English README](README.md)

## 一言でいうと

多くのスタートアップを素早く見る投資家のために、散らかったメモを「証拠で上限をかけた投資メモ」に変え、速さが過信に変わらないようにするツールです。

## 誰のためのものか

スカウト、エンジェル投資家、アクセラレーター、初期投資家のためのワークフローです。

## 何が困りごとか

初期デューデリジェンスでは、Webサイト、創業者の主張、市場メモ、通話メモがばらばらに集まります。AIはきれいなメモをすぐ作れますが、証拠、仮説、リスク、人間の判断を混ぜてしまう危険があります。

## どう解決するか

- 投資可能性を「仮説」として表示します。
- 証拠が薄い時はスコアに上限をかけます。
- 証拠、仮説、リスク、次に聞く質問を分けます。
- Markdownの投資メモとして出力できます。
- Geminiの動作証拠は、秘密情報を含まない形で表示します。

## はじめての使い方

1. 右上の `日本語` を押します。
2. 左側に会社名、市場、調査メモ、証拠の種類を入れます。
3. `投資メモを作る` を押します。
4. 右側の `投資スナップショット` で大まかな見立てを見ます。
5. `リスクマップ` と `次に聞く投資家質問` を見て、次の面談で確認することを決めます。
6. 必要なら `Markdownメモをコピー` で、人間が確認する投資メモとして使います。

## 画面の見方

- スコアは最終判断ではありません。証拠の強さに応じた作業仮説です。
- `証拠メモ` は、入力メモから拾えた根拠です。
- `確認すべき仮説` は、まだ本当か分からない前提です。
- `リスクマップ` は、見落とすと危ない点です。
- `AI動作証拠` は、Geminiが実際に動いたことだけを示します。売上や顧客検証を証明するものではありません。

## 日本語対応

画面右上の `日本語` ボタンで、日本語UIに切り替えられます。英語圏の審査員にも、日本語話者にも伝わるように、英語提出を維持しながら日本語導線を入れています。

## 主張の境界

これは投資助言ではありません。売上、顧客検証、ライブWeb抽出、最終投資判断を証明するものではありません。証拠が添付されていない部分は、あくまで調査仮説として扱います。

## 検証

```bash
npm run verify
```

検証では、英語UI、証拠スコアの安全装置、日本語UI切替、デモ動画、no-secrets確認をまとめて確認します。
